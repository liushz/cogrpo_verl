问题：训练在完成2个步骤后完全卡住，无法继续

根本原因：
  • vLLM 0.12.0 的 LLM 类不支持运行时动态 LoRA 管理
  • 没有 add_lora() 方法
  • Verifier LoRA 更新后保存成功，但无法重新加载到 vLLM 推理引擎
  • 代码假设 add_lora 存在，导致每次重载都失败

证据：
  • 日志: "WARNING: vLLM engine does not support add_lora; cannot reload verifier LoRA"
  • 训练在 11:20 最后一次日志后就完全停止
  • Verifier 更新在正常进行，但更新后的权重无法同步回 rollout

影响：
  ✅ Verifier LoRA 初始加载成功
  ✅ Verifier Intervention 正常工作（每样本1次干预）
  ✅ Verifier Reward 计算正确
  ✅ Verifier 更新正在执行（学习率 5e-07 → 1e-06）
  ✅ Verifier LoRA 保存成功
  ❌ Verifier LoRA 无法重新加载 ← 卡点

文件：verl/workers/rollout/vllm_rollout/vllm_rollout_spmd.py
函数：reload_verifier_lora()

修改内容：
  1. API 自动检测
     • 使用 inspect.signature() 检测 add_lora 的参数类型
     • 自动判断是 AsyncLLMEngine API 还是 legacy API

  2. 正确的 LoRARequest 使用
     from vllm.lora.request import LoRARequest
     lora_req = LoRARequest(
         lora_name="verifier_lora",
         lora_int_id=prev_lora_int_id,
         lora_path=verifier_lora_path
     )
     success = engine.add_lora(lora_request=lora_req)

  3. 改进的错误处理和日志
     • 详细的成功/失败日志
     • 优雅的 fallback 机制
